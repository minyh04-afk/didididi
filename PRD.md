## 1. 개요

### 제품명
**방탈출 커뮤니티 플랫폼** (코드명: "ExitCom")

### 목표
방탈출 취미자들이 스포일러 없이 좋은 테마를 발견하고, 팀원을 쉽게 찾아 함께 경험하는 원스탑 플랫폼

### 비전
> "방탈출은 혼자하는 게 아니라, 함께하는 취미다"

### 타겟 사용자

**1차 (핵심)**
- 월 2회 이상 방탈출 경험자
- 신뢰도 높은 정보 추천을 원하는 사람
- 팀원 구성 시간을 줄이고 싶은 사람
- 페르소나: 박준호 (개인, 40회 경험), 김지은 (커플, 30회 경험)

**2차 (확장)**
- 정기 팀을 유지하는 커플/그룹
- 패시브 유저 (파트너에 의존)
- 페르소나: 이준영 (커플, 30회 경험)

### 시장 규모
- 국내 방탈출 업체: ~300개
- 월 1회 이상 경험자: ~200만명
- 월 2회 이상 경험자(우리 타겟): ~50만명

### 성공 메트릭
- 월활성 사용자(MAU): 10,000명 (6개월)
- 팀 매칭 성공률: 70% 이상
- 리뷰 게시율: 30% 이상 (경험자 기준)
- 스포일러 필터 사용률: 80% 이상

## 2. 문제 정의

### 근본 문제
방탈출 취미자들은 **3가지 동시 문제**를 겪고 있음

#### ① 테마 발견 효율성 부족
- **기존 방식**: 인스타 + 네이버 + 구글맵을 돌아다니며 정보 수집
- **신뢰도**: 낮음 (리뷰 출처 불명확)
- **소요시간**: 주 2-3시간
- **결과**: 선택 피로, 낚임 경험, 돈과 시간 손실

#### ② 스포일러 피하기 불가능
- **문제**: 인기 테마는 3개월이면 유튜브 풀이가 나옴
- **현상**: 정보를 찾으려 해도 자동 추천으로 스포일러 노출
- **손실**: 기대감 50% 이상 감소, 경험 만족도 저하
- **반복**: 새 테마를 계속 찾아야 함 → 테마 고갈

#### ③ 팀원 구성의 어려움
- **문제**: 레벨 + 시간대 + 지역이 모두 맞는 사람 찾기 어려움
- **현황**: 카톡 그룹에서 "누가 갈래?" 반복
- **비용**: 팀 구성 시간 = 주 3-4시간
- **결과**: 혼자 가거나, 실력 차이로 데이트 감점

### 데이터 근거
- **인터뷰 5명 중 5명**: "새 테마 부족"이 가장 큰 문제 언급
- **팀 구하기 시간**: 평균 4.8시간/주 (박준호 기준)
- **스포일러 영향**: 경험 만족도 30% 감소 (김지은)
- **입력 의지**: "기록 시스템은 사용 안 함" (인터뷰 5/5)

### Root Cause (5 Whys 분석)
**방탈출은 본질적으로 "일회용 콘텐츠"**
- 한 번 알게 된 퍼즐/답은 재미의 80% 사라짐
- 지적재산권 보호 불가능 (스포일러 확산 자동화)
- 공급자는 창의적 개발 비용 부담, 소비자는 3-6개월 수명만 경험
- 결과: 테마 고갈 + 비용 부담 + 팀 구성 어려움 악순환

## 3. 솔루션

### 핵심 솔루션
> **"스포일러 필터링 + 팀 매칭" 일원화 플랫폼**

### 3가지 핵심 기능

#### ① 스포일러 필터 리뷰 시스템

**작동 원리:**
- 리뷰 작성시 [스포일러 포함 여부] 필수 선택
- 사용자는 [스포일러X만 보기] 필터로 필터링
- AI + 커뮤니티 신고로 거짓 방지

**효과:**
- 신뢰도 높은 정보만 노출
- 기대감 유지 (스포일러 회피)
- 정보 신뢰도: 기존 3.2점 → 4.5점 (40% 상승)
- 낚임 감소: 실패율 20% → 5%

**구현 세부:**
```
리뷰 작성:
  1. 점수 선택 (1~5점)
  2. 리뷰 텍스트 작성
  3. [스포일러 포함 여부] 필수 선택
     ☐ 스포일러 없음 (난이도/분위기만)
     ☑ 스포일러 있음 (결말/트릭 포함)
  4. 게시

리뷰 조회:
  1. [스포일러X만 보기] 필터 활성
  2. 필터링된 리뷰만 표시
  3. 비활성시 모든 리뷰 보여줌
     (단, 스포일러O는 블러 처리)
```

#### ② 팀 매칭 시스템

**작동 원리:**
- 사용자가 "이날 이 난이도 갈래"로 스케줄 공개
- 매칭 알고리즘이 1분 내 제안
- Chat 기반 확정

**효과:**
- 팀 구성 시간: 주 4시간 → 30분 (87% 감소)
- 신뢰도 높은 팀원 발견 (경력, 팔로워 기반)
- 네트워크 효과 (팀원 추천 체인)

**구현 세부:**
```
스케줄 공개:
  1. [팀 매칭] 탭에서 [+ 스케줄]
  2. 입력: 날짜, 시간대, 인원수, 지역, 난이도, 선호 장르
  3. [매칭 시작] 클릭

매칭 알고리즘:
  - 필터: 날짜 ∩ 시간 ∩ 지역 ∩ 난이도 일치
  - 정렬: 신뢰도 점수 (높은순)
  - 제안: 상위 5명, 1분 이내

확정:
  1. 제안된 팀원 프로필 보기
  2. [참여하기] → Chat 시작
  3. 일정 확인 후 [확정하기]
```

#### ③ 신뢰도 점수 시스템

**계산 공식:**
```
신뢰도 = (별점 × 리뷰유형가중치) + (팔로워수보너스) + (경력보너스)

세부:
① 리뷰 유형 가중치:
   - 스포일러X: 1.2배
   - 스포일러O: 0.8배
   
② 팔로워 수 보너스:
   - 0-10명: +0점
   - 11-50명: +0.1점
   - 51-100명: +0.2점
   - 100명+: +0.3점
   
③ 경력 보너스 (경험한 방탈출 수):
   - 1-10회: +0점
   - 11-30회: +0.1점
   - 31-50회: +0.2점
   - 50회+: +0.3점
```

**예시:**
```
"지은이 (4점 리뷰, 스포일러X, 팔로워 47명, 30회 경험)"
= (4 × 1.2) + 0.1 + 0.1 = 4.9점

"익명이 (5점 리뷰, 스포일러O, 팔로워 0명, 1회)"
= (5 × 0.8) + 0 + 0 = 4.0점
```

**효과:**
- 정보 신뢰도 판단 시간: 5분 → 5초 (60배)
- 투명한 점수로 정보 신뢰도 판단 용이
- 사기 리뷰 자동 감지 (점수 이상 현상)

## 5. 기술 스택

### Frontend
```
React Native (iOS + Android)
또는 Flutter (성능 최적화)

라이브러리:
- Navigation: React Navigation
- State Management: Redux / Zustand
- UI: React Native Paper
- HTTP: Axios + React Query
```

### Backend
```
Node.js + Express (또는 Python + FastAPI)

구조:
- API Server: Express.js
- Real-time: Socket.io (Chat)
- Job Queue: Bull (배경 작업)
- Caching: Redis
```

### Database
```
Primary: PostgreSQL
- 사용자, 리뷰, 팀매칭, 메시지 (관계형)

Cache: Redis
- 실시간 Chat 메시지
- 매칭 큐
- 알림 저장

Search: Elasticsearch
- 테마 검색 최적화
- 리뷰 전문 검색
```

### 외부 API
```
- 카톡 로그인: Kakao OAuth 2.0
- 지도: Google Maps API
- 푸시 알림: Firebase Cloud Messaging
- 애널리틱스: Google Analytics + Mixpanel
```

### 배포
```
클라우드: AWS
- 컴퓨팅: EC2 (t3.medium)
- 데이터베이스: RDS (PostgreSQL)
- 캐시: ElastiCache (Redis)
- 파일 저장: S3
- CDN: CloudFront

CI/CD: GitHub Actions
- 테스트 자동화
- 릴리즈 자동화
```

## 6. 성공 지표

### 주요 지표 (OKR)

#### ① 사용자 성장 (Growth)
```
목표: 6개월 내 10,000 MAU

마일스톤:
- 1개월: 500 가입자
- 3개월: 3,000 MAU
- 6개월: 10,000 MAU

측정:
- Google Analytics: 일일 활성 사용자
- Firebase: 설치 및 가입 유입
- 데이터베이스: 월별 활성 사용자 집계
```

#### ② 팀 매칭 성공률 (Engagement)
```
목표: 70% 이상

마일스톤:
- 런칭 후 2주: 30%
- 1개월: 50%
- 3개월: 70%

측정:
- 매칭 요청 수 / 수락 수
- 확정된 팀 매칭 / 전체 요청
- 팀 매칭 후 플레이 실제 여부 (Survey)
```

#### ③ 리뷰 게시율 (Content)
```
목표: 경험자의 30% 이상이 리뷰 작성

마일스톤:
- 1개월: 10%
- 3개월: 20%
- 6개월: 30%

측정:
- (리뷰 수) / (My List 경험 수)
- 리뷰 작성 사용자 / 활성 사용자
- 평균 리뷰 길이 (quality check)
```

#### ④ 스포일러 필터 효과 (Quality)
```
목표: 스포일러X 리뷰의 만족도 4.5 이상

측정:
- 리뷰별 만족도 평점 (1~5점 추가 선택)
- 스포일러O vs X 비교

기준:
- 스포일러O 평균: 3.2점
- 스포일러X 평균: 4.5점 (목표)
```

#### ⑤ DAU / MAU 비율 (Retention)
```
목표: 30% 이상

의미:
월 3회 이상 활동하는 충성사용자

마일스톤:
- 1개월: 15%
- 3개월: 25%
- 6개월: 30%

측정:
- DAU / MAU × 100
```

```

```

**목표 사용자**: 3,000 MAU

### Phase 4 (6개월 이후): 수익화 & 생태계

**수익화 경로:**

1. **광고 수익** (40% 예상)
   - 시설/상품 프로모션
   - 배너 광고 (Discover 상단)

2. **구독료** (30% 예상)
   - 프리미엄: "찜한 세트 예약", 광고 제거
   - 시설 운영자 대시보드

3. **수수료** (30% 예상)
   - 앱 내 예약 수수료 (5%)
   - 팀 여행 상품 판매

**생태계 확장:**
- 방탈출 → 보드게임 바 → 스포츠 등 취미 플랫폼화
- 팀 기반 모든 취미 활동으로 확장 가능

## 8. 부록

### A. 서비스 정책

#### 스포일러 관리 정책

```
정의:
- 스포일러O: 테마 결말/핵심 트릭 노출
- 스포일러X: 난이도, 분위기만 언급

작성자 책임:
① 리뷰 작성시 [스포일러 포함 여부] 필수 선택
② 스포일러O 체크시 콘텐츠 블러 처리
③ 거짓 선택시 커뮤니티 신고 가능

사용자 권리:
① [스포일러X만 보기] 필터 사용 가능
② 스포일러O 리뷰는 자동 숨김 (펼치기 옵션)
③ 블록 후 해당 리뷰 영구 숨김

처벌:
- 거짓 스포일러 신고 5회: 리뷰 게시 제한
- 자동 스포일러 체크 불가: 관리자 검수 필수
```

#### 팀 매칭 정책

```
신청 및 취소:
① 스케줄 공개 후 매칭 요청
② 수락 후 Chat 시작
③ 예약 전까지 취소 가능 (페널티 없음)

불성실 처벌:
① 예약 24시간 전 취소: 자유
② 24시간 이내 취소: 신뢰도 -1
③ 약속시간 미참여 (연락 없음): 신뢰도 -3
④ 3회 누적시 팀 매칭 제한 (7일~1개월)

신뢰도 복구:
① 5회 성공 참여시 -1 복구
② 팀원 추천장 5개 받으면 -2 복구
```

#### 커뮤니티 가이드

```
금지사항:
- 광고/영리 목적 리뷰
- 욕설/비하/개인정보 노출
- 스포일러 의도적 노출
- 테마 제작사/운영사 사칭

처벌 단계:
1단계: 경고 (첫 위반)
2단계: 댓글 기능 제한 (7일)
3단계: 계정 일시정지 (30일)
4단계: 계정 영구삭제
```

### B. 정보 구조 (IA)

```
메인 탭:
├─ Discover (탐색)
│  ├─ 검색 + 필터
│  ├─ [스포일러X만 보기]
│  └─ 테마 카드 (평점, 신뢰도)
│
├─ Team (팀 매칭)
│  ├─ 내 스케줄
│  ├─ 매칭 제안 (실시간)
│  └─ Chat
│
├─ My List (기록)
│  ├─ 다녀온 곳 (달력)
│  ├─ 점수 & 리뷰 작성
│  └─ 통계
│
├─ Message (메시지)
│  └─ 팀원과 소통
│
└─ Profile (프로필)
   ├─ 개인정보 + 통계
   ├─ 뱃지 & 신뢰도
   └─ 설정
```

### C. 사용자 여정 (User Journey)

#### 박준호 (개인, 적극적)
```
Step 1. 불안감 → "다음주에 어디 갈까"
Step 2. 발견 → 앱 검색
Step 3. 진입 → 회원가입 (카톡 간편)
Step 4. 탐색 → Discover 필터링
Step 5. 선택 → 테마 상세 보기
Step 6. 매칭 → 팀원 제안
Step 7. 소통 → Chat으로 일정 확인
Step 8. 예약 → 외부 링크로 예약
Step 9. 경험 → 테마 플레이
Step 10. 리뷰 → "스포일러 없음" 체크하고 게시
Step 11. 반복 → 다음 테마 검색
```

#### 김지은 (커플, 주도적)
```
Step 1. 스트레스 → "또 어디갈까" + SNS 검색
Step 2. 발견 → 친구 추천
Step 3. 진입 → 카톡 로그인
Step 4. 탐색 → [스포일서X만 보기] 필터 활용
Step 5. 신뢰 → 비스포일러 리뷰 확인
Step 6. 매칭 → 팀원 자동 제안
Step 7. 소통 → Chat으로 스케줄 조율
Step 8. 예약 → 확정
Step 9. 경험 → 테마 플레이 + 인증샷
Step 10. 리뷰 → "스포일러 없음" 강조하며 게시
Step 11. 공유 → 인스타 스토리 업로드
Step 12. 반복 → 다음달 예약 준비
```

#### 이준영 (커플, 패시브)
```
Step 1. 의존 → 지은이가 정해준 일정 확인 (Chat)
Step 2. 확인 → "어디 가기로 했었지?" (My List)
Step 3. 탐색 → 지은이의 리뷰 보기
Step 4. 예약 → 지은이가 이미 완료
Step 5. 경험 → 테마 플레이
Step 6. 피드백 → "괜찮았어" (지은이에게 구두)
Step 7. 다음 → 지은이 지시 대기
```
